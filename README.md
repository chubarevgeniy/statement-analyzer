# statement-analyzer

Локальный анализатор банковских выписок (PDF). Работает **полностью в браузере**: ни один
файл и ни одна транзакция не отправляются на сервер. В коде **нет никаких личных данных** —
IBAN, имена и операции распознаются из самих выписок при загрузке и хранятся только в
IndexedDB вашего браузера. Поэтому приложение можно свободно пересылать другим людям.

## Возможности
- Импорт PDF-выписок **Deutsche Bank**, **Trade Republic** и **Revolut** (в т.ч. мультивалютных).
- Догрузка выписок помесячно без дублей: повторная загрузка сообщает «эти данные уже есть».
- Категоризация: неизвестные операции предлагается отнести к существующей категории или
  создать новую; выбор запоминается для будущих импортов.
- Автоопределение **внутренних переводов между своими счетами** — они не влияют на доход/расход.
- **Профили (люди):** счета группируются по владельцу. В дашборде «палочками» выбираете, чьи
  данные смотреть; выбор нескольких людей считается домохозяйством (переводы между ними неттятся).
- Конвертация не-EUR операций в EUR: для Revolut берётся готовый EUR-эквивалент из выписки;
  иначе курс подтягивается из бесплатного API ([frankfurter.app](https://www.frankfurter.app),
  ECB) и кэшируется, либо вводится вручную.
- Дашборд: заработано, потрачено, накоплено, вложено в инвестиции; разбивка по категориям и
  динамика по месяцам. Покупки ETF в Trade Republic считаются накоплениями, дивиденды и
  проценты — доходом.
- Исключение любых категорий из подсчётов (по умолчанию исключены внутренние переводы и
  накопления/инвестиции; можно исключить, например, внос наличных).
- Две независимые базы данных в IndexedDB (транзакции и категории/маппинги) — со сбросом каждой
  по отдельности.

## Запуск
```bash
npm install
npm run dev      # дев-сервер на http://localhost:5173
```

## Сборка и тесты
```bash
npm run build    # продакшн-сборка в dist/ (можно открыть как статические файлы)
npm run test     # юнит-тесты парсеров и аналитики
```

## Архитектура
- `src/parsers/` — извлечение текста из PDF (pdf.js) и парсеры по банкам.
- `src/db/` — две базы IndexedDB (`transactions` и `categories`).
- `src/services/` — импорт, дедупликация, курсы валют, внутренние переводы, аналитика.
- `src/components/` — UI (импорт, дашборд, транзакции, настройки).

Тестовые данные в репозитории **синтетические**; реальные выписки в код не попадают.
